# BigQuery カラム値定義

## orders テーブル

### order_status
| 値 | 意味 | ビジネス用語 |
|----|------|-------------|
| 1 | 受注済み | 新規注文 |
| 2 | 出荷準備中 | 処理中 |
| 3 | 出荷済み | 配送中 |
| 4 | 配達完了 | 完了 |
| 9 | キャンセル | キャンセル |

**よく使う条件:**
- 完了注文: `order_status = 4`
- キャンセル除外: `order_status != 9`
- アクティブ注文: `order_status IN (1, 2, 3)`

## users テーブル

### user_type
| 値 | 意味 |
|----|------|
| 1 | 一般会員 |
| 2 | プレミアム会員 |
| 3 | 法人会員 |

**よく使う条件:**
- プレミアム会員: `user_type = 2`
- 個人会員（一般+プレミアム）: `user_type IN (1, 2)`
